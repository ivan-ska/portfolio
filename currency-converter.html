Currency Converter

This project started as a personal solution to a daily annoyance. I was tired of googling exchange rates, managing messy browser tabs, and dealing with ad-heavy converter apps. I wanted something clean, instant, and capable of tracking multiple currencies at once without the clutter.

My curiosity led me to try solving this problem by myself. Using knowledge of UX/UI and vibe coding, I implemented an open-source custom currency converter that can be easily adapted to anyone's needs.

My first priority was proving the core logic. This initial prototype was intentionally rough: awkwardly positioned controls, wierd copy, and the overall structure felt disjointed. There was no way to reset the values—even refreshing the page didn’t clear the data.

But it was fully functional, fetched real-time API data and performed accurate conversions. For a first vibecoding experiment, seeing the logic hold up was a massive "proof of concept" win.

Progressive iterations helped strip away the clutter and focus on real usability. Once the vision for a polished, custom UI became clear, I took the project to Figma to build a design system and map out every detail before the final build.

Iterating in Figma allowed me to refine both the visual language and the core functionality. This stage helped me realize that the "+" and "-" buttons were rarely used and only cluttered the interface. However, the "X" for resetting was essential, especially for the mobile version.

By building a custom library I moved away from generic assets in favor of a distinctive yet user-friendly look. The goal was to create a tool that offered a seamless experience for completing a simple task.

The final web version mirrored the Figma design perfectly, with every interaction working as intended. All the logic and visuals finally came together into a smooth, production-ready tool. Satisfied with the result, I decided to take it one step further and transform the site into a dedicated Web App for a more native experience.

Using vibe-coding, I implemented Web App functionality so the converter could live right on my iPhone home screen. It now opens with a custom icon and functions without a browser bar, behaving exactly like a native application. This journey from a rough draft to a daily-use tool was a great reminder that a designer's curiosity can turn any personal frustration into a functional reality.
